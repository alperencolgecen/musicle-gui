# Contributing to MusicLe

Thank you for your interest in contributing to MusicLe!

## How to Contribute

1. **Fork** the repository.
2. **Create a feature branch** (`git checkout -b feature/your-feature`).
3. **Make your changes** — keep them focused and well-documented.
4. **Test your changes** by building the project with CMake and running the application.
5. **Commit** your changes with a clear and concise commit message.
6. **Push** to your fork and open a **Pull Request**.

## Code Style

- Follow modern C++20 conventions.
- Use consistent naming and formatting (match existing code).
- Keep functions small and focused.

## Reporting Issues

If you find a bug or have a feature request, please open an issue using the provided issue templates.

## Pull Request Checklist

- [ ] Builds successfully with Visual Studio 2022
- [ ] Follows existing code style
- [ ] Documentation updated if needed
- [ ] Commits are squashed and meaningful
